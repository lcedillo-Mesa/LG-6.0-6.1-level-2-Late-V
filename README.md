# Assessment 2 — CSS Selectors & Box Model

---

## Your Task

You will be given a rendered mockup of a portfolio page.

Your job is to write the CSS rules that make your page look as close to the mockup as possible.

Open `styles.css` and write your rules below the line that says `/* Write your code below this line */`.

---

## What you need to demonstrate

**Selectors (LG 6.0)**
- Use class selectors to target individual elements
- Use an ID selector on at least one element
- Use tag selectors where appropriate
- You will need to decide which selector type makes sense for each element

**Box model (LG 6.1)**
- Use margin, padding, border, and border-box together to style at least one component
- Your styling should show that you understand how these properties interact with each other

---

## Tips

- Look closely at the mockup before you start writing — notice which elements share the same style and which ones are unique
- If two elements look the same, they probably share a class
- If one element looks unique, it probably has an ID
- Check your work in the browser as you go — you do not need to finish everything before previewing

## Rubric

### LG 6.0 — CSS Selectors

| Level | What this looks like |
|-------|-------------|
| **4 — Exceeds** | Your page matches the mockup. You used tag selectors, class selectors, and at least one ID selector. Your ID is only used once in the HTML. Your classes are used on elements that share the same style. You added at least one CSS comment explaining why you chose a specific selector type for an element. |
| **3 — Meets** | Your page looks very close to the mockup. You used at least one class selector, at least one ID selector, and at least one tag selector. Each selector is targeting the correct element. |
| **2 — Approaching** | Your page partially matches the mockup. You used class selectors but no ID selector, or you used an ID selector on more than one element. Some elements are styled but others are missing. |
| **1 — No attempt** | Your stylesheet has no class or ID selectors. Nothing on the page has been styled beyond the setup code. |

---

### LG 6.1 — Box Model

| Level | What this looks like |
|-------|-------------|
| **4 — Exceeds** | Your page matches the mockup. You used margin, padding, border, and border-box together on at least one component. You also applied box model properties to at least one other element that was not specifically asked for, improving the spacing or layout of the page. |
| **3 — Meets** | Your page looks very close to the mockup. You used at least 2–4 box model properties together on a component — for example margin and padding together, or border and padding together. The spacing and sizing on that component matches what you see in the mockup. |
| **2 — Approaching** | You used at least one box model property but only on its own — for example just padding, or just a border, without combining them. The component is partially styled but does not fully match the mockup. |
| **1 — No attempt** | No box model properties have been written beyond the setup code. |

---

## What does level 4 look like in practice?

**For selectors:** your CSS file has rules using all three selector types. At least one rule has a comment above it that says something like `/* using an ID here because this element only appears once */`. Your ID selector is only on one element in the HTML.

**For the box model:** at least one component uses margin, padding, border, and border-box all in the same rule. You also went back and added box model properties to another element — like adding padding to the header or margin to the footer — that makes the overall page look more polished than the mockup.
